# OryxLab-Pro 윈도우 실행 가이드

OryxLab-Pro 예약 시스템을 윈도우 환경에서 쉽게 실행하기 위한 가이드입니다.

## 1. 사전 준비 (필수)
이 프로젝트를 실행하려면 **Python**이 설치되어 있어야 합니다. (이미 설치되어 있다면 이 단계는 건너뛰세요)

1. [Python 다운로드 페이지](https://www.python.org/downloads/) 접속
2. **Python 3.9** 이상의 버전을 다운로드 및 설치
3. 설치 화면 하단의 **"Add Python to PATH"** 체크박스를 **반드시** 선택해주세요.

## 2. 초기 설정
`setup_env.bat` 파일을 더블 클릭하여 실행하세요.
- 자동으로 가상 환경(`.venv`)을 만들고 필요한 라이브러리를 설치합니다.
- "설정이 완료되었습니다" 메시지가 나오면 창을 닫아도 됩니다.

## 3. 서버 실행
`run_app.bat` 파일을 더블 클릭하여 실행하세요.
- 검은색 창이 열리고 서버가 시작됩니다.
- 웹 브라우저를 열고 `http://127.0.0.1:5000` 주소로 접속하면 예약 시스템을 사용할 수 있습니다.

## 4. 관리자 로그인
- 주소: `http://127.0.0.1:5000/login`
- 기본 비밀번호: `admin123!`
  - 로그인 후 설정에서 변경할 수 있습니다.

## 💾 데이터 백업 및 NAS 이전 가이드 (중요)

이 시스템은 모든 데이터를 **파일 1~2개**에 보관하므로 관리가 매우 쉽습니다.
나중에 Synology NAS 등으로 서버를 옮길 때 아래 파일만 챙기면 모든 데이터(예약, 설정, 차단 내역 등)가 그대로 보존됩니다.

### 1. 핵심 데이터 파일 위치
프로젝트 폴더 내 `instance` 폴더 안에 있는 다음 파일들이 핵심입니다.
- `instance/library.db` : **(가장 중요)** 예약 명단, 관리자 비밀번호, 설정, 차단 목록 등 모든 핵심 데이터
- `instance/logs.db` : 접속 로그 및 서버 에러 로그 (없어도 작동은 하지만 기록 보관용)

### 2. Synology NAS / Docker로 이전 시
나중에 NAS의 Docker 기능을 사용하여 서버를 구축할 경우, 데이터 유실을 막기 위해 **볼륨 마운트(Volume Mount)** 설정을 해야 합니다.

**Docker 실행 시 예시 옵션:**
```bash
-v /volume1/docker/oryx_lab/instance:/app/instance
```
*   설명: NAS의 `/volume1/docker/oryx_lab/instance` 폴더를 컨테이너 내부의 `/app/instance` 와 연결한다는 뜻입니다.
*   이렇게 하면 컨테이너를 지웠다 다시 만들어도 데이터는 NAS에 안전하게 남아있습니다.

### 3. 수동 백업 방법
관리자 페이지의 **[백업 및 관리]** 탭에서 `DB 파일 다운로드 (.sqlite)` 버튼을 누르면 `library.db` 파일을 즉시 다운로드하여 PC에 보관할 수 있습니다. 주기적으로 다운로드해두시는 것을 권장합니다.
